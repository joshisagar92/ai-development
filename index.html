Test-Driven Development workflow (Red-Green-Refactor), code quality checklist, refactoring triggers, and YAGNI principles.

Key Rules: Write failing test first, methods <15 lines, max 3 parameters, extract duplicated code
Codex | Claude

🔒 Security Practices

OWASP Standards

Security implementation based on OWASP. SQL injection prevention, XSS encoding, access control verification, and password storage.

Key Rules: Parameterized queries, output encoding, object ownership, bcrypt/Argon2 hashing
Codex | Claude

🔬 Testing Practices

Test Patterns

Test pyramid, AAA structure (Arrange-Act-Assert), test doubles (stubs, mocks, fakes), and test data builders.

Key Rules: Tests <100ms, no external dependencies, one assertion per test, deterministic
Codex | Claude

🗃 Database Practices

SQL & Migrations

SQL best practices, migration patterns, indexing strategy, transaction patterns, and connection pool configuration.

Key Rules: Version all schemas, parameterized queries, index WHERE/JOIN columns, use transactions
Codex | Claude

🔗 Integration Practices

APIs & Services

API design (Richardson Maturity Model), RESTful patterns, circuit breaker, retry with backoff, correlation IDs, and versioning.

Key Patterns: Tolerant Reader, Circuit Breaker, Idempotent Receiver, API versioning
Codex | Claude

🛠 Design Practices

SOLID & Patterns

SOLID principles, Tell Don't Ask, Law of Demeter, Four Rules of Simple Design, and common patterns (Repository, Factory, Strategy).

Anti-Patterns: Anemic Domain Model, God Class, Premature Abstraction, Speculative Generality
Codex | Claude
